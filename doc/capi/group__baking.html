Detailed Description

Functions for baking sound propagation information into acoustic probes.

Baking allows detailed acoustic responses to be calculated at design time (just like precomputed lighting), resulting in higher quality sound propagation effects, reduced CPU usage, but increased memory and disk space usage. Phonon lets you bake parametric or convolution reverb at each probe, as well as source-to-listener sound propagation effects from various sources.

A callback that is called to update the application on the progress of the iplBakeReverb or iplBakePropagation functions.

You can use this to provide visual feedback to the user, like a progress bar.

Bakes reverb at all probes in a Probe Box.

Phonon defines reverb as the indirect sound received at a probe when a source is placed at the probe's location. This is a time-consuming operation, and should typically be called from the game engine's editor. The numThreads set on the IPLSimulationSettings structure passed when calling iplCreateEnvironment to create the Environment object are used for multi-threaded baking.

Parameters
environmentHandle to an Environment object.
probeBoxHandle to the Probe Box containing the probes for which to bake reverb.
bakingSettingsThe kind of acoustic responses to bake.
progressCallbackPointer to a function that reports the percentage of this function's work that has been completed. May be NULL.

Bakes propagation effects from a specified source to all probes in a Probe Box.

Sources are defined in terms of a position and a sphere of influence; all probes in the Probe Box that lie within the sphere of influence are processed by this function. This is a time-consuming operation, and should typically be called from the game engine's editor. The numThreads set on the IPLSimulationSettings structure passed when calling iplCreateEnvironment to create the Environment object are used for multi-threaded baking.

Parameters
environmentHandle to an Environment object.
probeBoxHandle to the Probe Box containing the probes for which to bake reverb.
sourceInfluenceSphere defined by the source position (at its center) and its radius of influence.
sourceIdentifierIdentifier of the source. At run-time, a Convolution Effect object can use this identifier to look up the correct impulse response information.
bakingSettingsThe kind of acoustic responses to bake.
progressCallbackPointer to a function that reports the percentage of this function's work that has been completed. May be NULL.

Bakes propagation effects from all probes in a Probe Box to a specified listener.

Listeners are defined solely by their position; their orientation may freely change at run-time. This is a time-consuming operation, and should typically be called from the game engine's editor. The numThreads set on the IPLSimulationSettings structure passed when calling iplCreateEnvironment to create the Environment object are used for multi-threaded baking.

Parameters
environmentHandle to an Environment object.
probeBoxHandle to the Probe Box containing the probes for which to bake reverb.
listenerInfluencePosition and influence radius of the listener.
listenerIdentifierIdentifier of the listener. At run-time, a Convolution Effect object can use this identifier to look up the correct impulse response information.
bakingSettingsThe kind of acoustic responses to bake.
progressCallbackPointer to a function that reports the percentage of this function's work that has been completed. May be NULL.

◆ iplCancelBake()

IPLAPI IPLvoid iplCancelBake ( )

Cancels any bake operations that may be in progress.

Typically, an application will call iplBakeReverb or iplBakePropagation in a separate thread from the editor's GUI thread, to keep the GUI responsive. This function can be called from the GUI thread to safely and prematurely terminate execution of any of these functions.
